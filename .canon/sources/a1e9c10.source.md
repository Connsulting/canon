# Canon Executable Release Automation

## Problem
Canon currently has no automated release pipeline that produces downloadable executables for each operating system. There is also no explicit version source in the repository.

## Goals
1. Add a repository version source starting at `v0.1.0`.
2. Expose version information from the CLI.
3. On tag push, build Canon executables for Windows, Linux, and macOS.
4. Create a GitHub release for the tag and attach the built executables.

## Scope
1. Add a `VERSION` file under `cmd/canon/`.
2. Add `canon version` output with support for `--version` and `-v`.
3. Add a GitHub Actions workflow triggered by semantic version tags.
4. Build and upload cross platform binaries as release assets.

## Requirements
1. `cmd/canon/VERSION` must be committed and initialized to `v0.1.0`.
2. `canon version` must print `canon <version>`.
3. `canon version --short` must print only `<version>`.
4. The release workflow must run on `push` for tags matching `v*`.
5. The workflow must fail if `VERSION` does not match the pushed tag.
6. The workflow must build binaries for:
   1. Linux amd64
   2. Windows amd64
   3. macOS amd64
7. Binary names must include version, OS, and architecture.
8. The workflow must create a GitHub release for the tag and upload all binaries.

## Validation
1. Unit tests must cover version command success and invalid argument behavior.
2. `go test ./...` and `go vet ./...` must pass.
3. Manual CLI positive test:
   1. `go run ./cmd/canon version`
4. Manual CLI negative test:
   1. `go run ./cmd/canon version invalid`
