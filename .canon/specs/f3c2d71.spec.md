## Problem
Canon conflict detection runs during `ingest` for one candidate spec against existing specs.
That protects linear workflows.

Git branch workflows create a gap.
Two branches can ingest independently, then merge cleanly at file level while still introducing semantic contradictions across spec content.
Canon currently has no post merge conflict scan to catch that contradiction class.

## Requested Behavior
1. Add `canon check` to scan canonical specs for semantic contradictions.
2. Use AI as the primary semantic adjudication engine for contradiction detection.
3. Make the command usable in local workflows and CI pipelines.

## Requirements
### Core
1. `canon check` must load specs from `.canon/specs/`.
2. The scan must compare all relevant spec pairs, not only latest ingest candidate.
3. Pair comparison must only run when domains overlap, using the existing overlap semantics.
4. Conflict matching must be AI adjudicated with structured JSON output.
5. Symmetric duplicates must not be reported twice.
6. If conflicts exist, command exit code must be non zero.
7. If no conflicts exist, command exit code must be zero.
8. Success output must be concise and include checked spec count plus conflict count.

### Output
1. Default output must be human readable.
2. Output must group conflicts by spec pair.
3. Each conflict row must include spec ids, spec titles, shared domain context, statement key, and contradictory lines.
4. Summary line must report total conflicts and total specs scanned.

### Flags
1. `--domain <name>` limits scan scope to specs in that domain.
2. `--spec <id>` checks one spec against the rest of the in scope set.
3. `--ai auto|from-response` controls AI check execution mode.
4. `--ai-provider codex|claude` chooses AI provider.
5. `--response-file <path>` reads precomputed AI JSON responses.
6. `--json` emits machine readable output for CI.
7. `--write` writes conflict reports under `.canon/conflict-reports/` using existing report format.

### JSON Contract
```json
{
  "passed": false,
  "total_specs": 12,
  "total_conflicts": 2,
  "conflicts": [
    {
      "spec_a": "b98ed16",
      "title_a": "AI Render Effective State Compression",
      "spec_b": "a3f7c01",
      "title_b": "Public API Access Policy",
      "domain": "api",
      "statement_key": "api rate limiting enforced",
      "line_a": "API rate limiting must be enforced on all endpoints",
      "line_b": "Public API endpoints must not enforce rate limiting"
    }
  ]
}
```

## Non Goals
1. Rule based semantic contradiction detection as the primary path.
2. Automatic conflict resolution.
3. Cross repository scans.
4. Replacing ingest time conflict checks.

## Validation
1. Unit tests for AI response parsing, pair normalization, and symmetric dedupe.
2. Unit tests for `--domain` and `--spec` filtering behavior.
3. Unit tests for JSON output contract.
4. CLI test where non conflicting specs return exit code zero.
5. CLI test where conflicting specs return non zero and readable conflict output.
6. CLI test where `--write` persists conflict report artifacts.
