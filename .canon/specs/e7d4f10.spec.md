# PII Exposure Scanner (Iteration 3 Draft)

## Problem
Run a repository-wide PII exposure audit with deterministic, reproducible checks across hardcoded literals, log/error leakage, env/secret surfaces, plaintext storage patterns, and `.gitignore` coverage.

## Scope
In scope:
1. Tracked repository files via `git ls-files` (excluding vendored/third-party paths).
2. Extra secret-sensitive surfaces in working tree (`.env*`, key/cert, credentials/secrets, dump-like files).
3. Code-level contextual checks for log/error interpolation and init-context ingestion risk.
4. Remediation for confirmed high-confidence risks.

Out of scope:
1. External services or runtime production data sources.
2. Binary forensic analysis.

## Success Criteria
1. Structured findings reported with: file, line, category, severity, detail, recommendation.
2. Totals summarized by category and severity.
3. Confirmed high-confidence issues remediated in code and/or ignore policy.
4. `go test ./...` and `go vet ./...` pass.
5. One positive and one negative CLI end-to-end command executed.

## Scan Commands
```bash
git ls-files | rg -v '^(vendor/|third_party/|node_modules/)'

# hardcoded literals
rg -n --pcre2 "\\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\\.[A-Za-z]{2,}\\b" ...
rg -n --pcre2 "\\b(?:\\+?1[-.\\s]?)?(?:\\([2-9][0-9]{2}\\)|[2-9][0-9]{2})[-.\\s][2-9][0-9]{2}[-.\\s][0-9]{4}\\b" ...
rg -n --pcre2 "\\b\\d{3}-\\d{2}-\\d{4}\\b" ...
rg -n --pcre2 "\\b(?:\\d{4}[- ]?){3}\\d{4}\\b|\\b\\d{13,19}\\b" ...
rg -n --pcre2 "\\b(?:25[0-5]|2[0-4]\\d|1?\\d?\\d)(?:\\.(?:25[0-5]|2[0-4]\\d|1?\\d?\\d)){3}\\b" ...
rg -n --pcre2 "\\b\\d{1,5}\\s+(?:[A-Za-z0-9.-]+\\s){0,5}(?:Street|St\\.?|Avenue|Ave\\.?|Road|Rd\\.?|Boulevard|Blvd\\.?|Lane|Ln\\.?|Drive|Dr\\.?|Way|Court|Ct\\.?)\\b" ...
rg -n --pcre2 "(?i)\\b(passport|driver'?s?\\s*license|drivers?\\s*licen[cs]e|license\\s*(?:number|no\\.?))\\b" ...

# logs/errors
rg -n --pcre2 "(fmt\\.(?:Sprintf|Errorf|Printf)|log\\.(?:Printf|Println|Fatalf|Fatal)|slog\\.[A-Za-z]+|zap\\.[A-Za-z]+|zerolog|logrus)[^\\n]*(email|name|phone|address|ssn|token|password|secret)" cmd internal
rg -n --pcre2 "(fmt\\.(?:Errorf|Sprintf|Printf)|errors\\.(?:Wrap|Wrapf))[^\\n]*(email|name|phone|address|ssn|token|password|secret)" cmd internal

# env/secrets and storage cues
find . -type f \( -name '.env' -o -name '.env.*' -o -name '*.pem' -o -name '*.key' -o -name 'credentials.*' -o -name 'secrets.*' -o -name '*.sql' -o -name '*.csv' -o -name '*.xlsx' \) -not -path './.git/*'
rg -n --pcre2 "(?i)(api[_-]?key|secret|token|password|passwd|client[_-]?secret|private[_-]?key|aws_access_key_id|aws_secret_access_key)\\s*[:=]"
```

# PII Exposure Scanner v3
## Desired Behavior
- `pii-scan --root <path>` emits human-readable structured findings by default.
- `pii-scan --json` emits stable snake_case JSON with `root`, `files_scanned`, `findings[]` (each: `rule_id`, `category`, `severity`, `file`, `line`, `detail`, `recommendation`), `summary` (`total_findings`, `highest_severity`, `findings_by_severity`, `findings_by_category`), `fail_on?`, `threshold_exceeded`.
- `pii-scan --fail-on <severity>` exits non-zero when highest finding severity meets or exceeds the threshold; JSON still emitted on stdout when paired with `--json`.
- Tracked-file enumeration uses `git ls-files`. Excluded path prefixes: `vendor/`, `third_party/`, `node_modules/`, `.git/`, `.canon/`.
- Output is deterministic across invocations; ordering is severity desc, category asc, file asc, line asc, rule_id asc.
3. Confirmed high-confidence issues remediated in code and/or `.gitignore` policy.
5. One positive and one negative CLI end-to-end command executed and parseable.
## Detector classes (initial iteration)
- Hardcoded literals: SSN-shaped (`\d{3}-\d{2}-\d{4}`), credit-card-shaped (`(?:\d{4}[- ]){3}\d{4}`), email literals.
- Secret-credential assignment patterns: case-insensitive match against `api[_-]?key`, `secret`, `token`, `password`, `client[_-]?secret`, `private[_-]?key`, `aws_access_key_id`, `aws_secret_access_key` followed by `:`/`=` and a value of >= 8 chars.
- Sensitive working-tree files: `.env*`, `*.pem`, `*.key`, `credentials.*`, `secrets.*`, dump-style (`*.sql`, `*.csv`, `*.xlsx`).
## Severity ladder
- Critical: SSN literals, credit-card literals, hardcoded secret-credential assignments.
- High: env files, `.pem`/`.key` files, `credentials.*` and `secrets.*` files.
- Medium: email literals, dump-style files (`.sql`, `.csv`, `.xlsx`).
- Low: reserved.
## Deferred to follow-up issues
- Phone-number, street-address, government-ID-keyword, IPv4 literal detection (high false-positive surface in code).
- Log/error-interpolation heuristic (needs AST awareness to avoid noise).
- `.gitignore` coverage cross-check (needs a parser pass).
