# PII Exposure Scanner Audit

## Scope and Method
Repository-wide audit on tracked files (`git ls-files`) with explicit checks for `.env*`, key/cert files, credentials/secrets files, and dump-like data artifacts. Vendored/third-party paths were excluded.

Checks run:
1. Hardcoded PII literals: email, phone, SSN, credit-card-like numbers, IP addresses, address literals, DOB patterns, passport/driver-license keywords, structured full-name patterns.
2. PII in logs/errors: `fmt.*`, `log.*`, `slog`, `zap`, `zerolog`, `logrus`, and wrapping/error contexts containing PII field names.
3. Env/secret surfaces: `.env*`, workflow/config files, plaintext secret assignment patterns.
4. Unencrypted storage cues: sensitive fields/columns and write paths likely to store plaintext sensitive data.
5. `.gitignore` gap analysis and tracked-file cross-check.

## Remediation Applied
1. Added sensitive-path exclusion in init project scanning and tests:
   - `internal/canon/init.go`
   - `internal/canon/init_test.go`
2. Hardened ignore policy for local secrets and dump-like artifacts:
   - `.gitignore`
